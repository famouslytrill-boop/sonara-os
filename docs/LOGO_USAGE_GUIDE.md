# Logo Usage Guide

## Clear Space

Keep at least one mark-width of empty space around each logo.

## Minimum Size

- Full logo: 160 px wide minimum.
- Mark only: 24 px minimum for UI, 32 px preferred.
- App icon: test at 16 px, 32 px, 64 px, and 192 px.

## Light Backgrounds

Use the full-color logo only on high-contrast light backgrounds. If contrast is weak, use a dark container or monochrome dark mark.

## Dark Backgrounds

Use token-driven color marks on dark slate backgrounds. Parent cyan and Launchpad amber should remain bright enough to pass contrast.

## Monochrome

Use monochrome for embossing, low-color documents, footer contexts, and small utility surfaces.

## Forbidden Usage

- Do not stretch.
- Do not rotate without an approved motion system.
- Do not recolor outside the token system.
- Do not place on low-contrast backgrounds.
- Do not use final/legal language before trademark review.
- Do not use the registered mark symbol unless registration exists.

## Parent vs Entity Logo

Use the parent logo for ownership, governance, security, docs, and shared infrastructure. Use the entity logo inside the entity dashboard, onboarding, settings, and app-specific routes.

## Launchpad Logo

Use the Launchpad mark only for the standalone Launchpad product surface. Do not imply guaranteed deployment or guaranteed launch success.
