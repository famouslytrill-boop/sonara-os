# Auto Update Policy

Allowed automatic updates: metadata refresh, star/fork/open issue counts, latest release info, last pushed date, archived status, recommended review status, admin recommendation drafts, Codex sprint drafts, and audit events.

Blocked automatic updates: dependency installs, source copying, production adapters, feature flag enablement, customer-facing integrations, external tool execution, billing/payment/security changes, merges, and deployments.
