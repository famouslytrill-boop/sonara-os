# Repo Review Workflow

1. Add a manual repository record or import metadata if `GITHUB_TOKEN` is configured server-side.
2. Categorize the repository and map it to Business Builder, Creator Studio, Growth Studio, or Shared Infrastructure.
3. Review license, security, privacy, maintenance, commercial use, and product fit.
4. Generate a Codex sprint draft only after review flags are understood.
5. Enable implementation only after owner/admin approval.
