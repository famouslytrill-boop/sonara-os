# SONARA Database Management Console

## Purpose

The founder/admin Database Management console provides a live, read-only inventory of the connected production Supabase PostgreSQL project.

Routes:

- `/admin/database-management`
- `/api/admin/database-management`
- `/admin/database` redirects to the unified console.
- `/admin/migrations` redirects to the Migrations section.

All routes require the existing founder/admin session middleware.

## Live sections

The console exposes the following database metadata without exposing credentials or arbitrary SQL execution:

1. Schema Visualizer
2. Tables
3. Functions
4. Triggers
5. Enumerated Types
6. Extensions
7. Indexes
8. Publications
9. Access Control
10. Policies
11. Roles
12. Configuration
13. Settings
14. Platform
15. Replication
16. Backups
17. Migrations

The data is generated by `public.sonara_database_management_snapshot()` and returned through the service-role-only Supabase RPC endpoint.

## Security contract

- The catalog RPC is revoked from `PUBLIC`, `anon`, and `authenticated`.
- Only `service_role` receives execute permission.
- Database credentials never enter the HTML or JSON response.
- Role password hashes, connection strings, provider keys, and subscription connection information are not selected.
- The web console cannot execute arbitrary SQL.
- The web console cannot create, alter, or drop schema objects.
- The web console cannot change roles, policies, publications, replication slots, or settings.
- The web console cannot create or delete backups.
- Every dashboard/API access is recorded through the existing administrator audit pipeline.

## Backup truthfulness

Supabase backup schedules, retention, restore points, and point-in-time recovery are provider-managed. PostgreSQL system catalogs do not provide a reliable view of those platform settings.

The Backups section therefore reports `provider_managed_verification_required` until the protected Supabase platform is reviewed. It must not display a fabricated success status.

## Migration reconciliation

The Migrations section compares:

- applied versions returned by `supabase_migrations.schema_migrations`;
- local SQL files in `supabase/migrations`;
- pending local versions;
- remote-only versions.

The console is synchronized only when every local migration version exists in remote migration history.

## Required migration

Apply:

```text
20260723033000_database_management_catalog.sql
```

Then verify:

```bash
pnpm run db:push
pnpm run verify:db
```

## API section filtering

The JSON endpoint accepts a read-only `section` query parameter:

```text
/api/admin/database-management?section=tables
/api/admin/database-management?section=policies
/api/admin/database-management?section=migrations
```

Invalid or absent section values return the complete snapshot.
